System Architecture & Logic

Project: Resume AI Shortlister

1. How It Works (System Flow)

The Resume AI Shortlister is an intelligent screening platform that automates the initial phase of recruitment using Generative AI. It follows a structured workflow to ensure accuracy and speed.

End-to-End Workflow:
  1. Upload: User provides a resume (PDF/TXT) via the React frontend.
  2. Extraction: Flask backend strips the text using PyPDF2.
  3. Inference: The text is sent to the Llama 3 model with a custom instructional prompt.
  4. Parsing: The AI outputs structured JSON data.
  5. Visualization: Results are displayed on a premium dashboard with color-coded metrics.

2. Shortlisting Logic & Thresholds

The system evaluates candidates based on three primary dimensions: Skill Alignment, Experience Depth, and Educational relevance. Instead of simple keyword matching, it performs a semantic analysis to understand the impact of previous work.

SHORTLISTING THRESHOLD

60 / 100

Candidates scoring 60 or above are automatically marked for shortlisting.

Scores are normalized to a 100-point scale. If the model provides a 1-10 rating, the system automatically scales it up by a factor of 10 to maintain consistency in reports.

3. Result Persistence

To ensure data is not lost between sessions, the application implements a local persistence layer using SQLite. Every analysis result is automatically logged.

What is stored?
  • Metadata: Filename and targeted job role.
  • Content: The raw text extracted from the resume.
  • Intelligence: The final score and the multi-dimensional AI analysis (strengths, weaknesses).

You can visualize all stored records by visiting the /api/resumes endpoint in your browser.

4. AI Model Description

The application leverages state-of-the-art Large Language Models (LLMs) to achieve human-like reasoning during the screening process.

  • Model Name: Meta Llama 3 (8B Instruct)
  • Deployment: Hosted via OpenRouter API
  • Optimization: Low temperature setting (0.1) for deterministic and reliable output.
  • Format: Strictly adheres to JSON output for seamless integration with the backend.
