ABSTRACT

In the highly competitive era of modern recruitment, Human Resource (HR) departments are inundated with an overwhelming volume of applications for every job opening. The traditional manual screening process is labor-intensive, time-consuming, and prone to human biases, often resulting in prolonged hiring cycles and the potential oversight of qualified candidates.

The Resume AI Shortlister is a cutting-edge web application designed to automate and optimize the initial resume screening process. Utilizing state-of-the-art Large Language Models (LLMs), specifically Meta's Llama 3 model, the system analyzes candidate resumes against specific job descriptions in real-time. Unlike traditional Applicant Tracking Systems (ATS) that rely on rigid keyword matching, this system leverages natural language understanding to evaluate candidates based on the semantic meaning of their skills, experience, and project work.

Users can upload resumes in PDF format and specify a target job role. The system parses the document, correlates the comprehensive profile with the requirements, and generates a detailed report card. This report includes a compatibility score (0-100), a definitive "Shortlisted" or "Not Shortlisted" status, and a qualitative analysis highlighting the candidate's strengths, weaknesses, and suggestions for improvement.

The project is implemented using a Flask (Python) backend for robust API handling and React.js for a responsive, modern frontend interface. The integration of Generative AI ensures that the evaluation is consistent, objective, and deeply analytical, significantly reducing the "Time to Hire" for organizations.

1. INTRODUCTION

1.1 Overview

Recruitment is a critical function of any organization, determining the quality of human capital that drives business success. However, the process of finding the right talent is fraught with challenges, primarily the "Volume vs. Quality" problem. A single job posting on a popular platform like LinkedIn or Indeed can attract hundreds, if not thousands, of applicants within days. For a recruiter, sifting through this mountain of documents to find the few "needles in the haystack" is a daunting task.

The Resume AI Shortlister project addresses this challenge by introducing Artificial Intelligence into the recruitment funnel. It serves as an intelligent assistant that pre-screens candidates, allowing human recruiters to focus their energy on interviewing and engaging with the top 10% of applicants rather than reading the bottom 90%.

1.2 Problem Statement

The traditional recruitment process suffers from several critical inefficiencies:

  • High Volume, Low Bandwidth: Recruiters spend an average of 6-7 seconds per resume (The Ladders, 2018). This rapid scanning leads to missed details and qualified candidates being overlooked.
  • Keyword Bias: Traditional ATS software adheres strictly to keyword matching. A candidate who lists "Software Engineering" might be rejected for a role requiring "Software Development" simply because the keywords don't match exactly, despite the roles being identical.
  • Unconscious Bias: Human recruiters may unconsciously bias their selection based on factors such as gender, university name, or gaps in employment, rather than pure skill alignment.
  • Fatigue: After reviewing 50 resumes, a recruiter's attention to detail significantly drops, leading to inconsistent evaluation criteria between the first and last applicant.

1.3 Objectives

The primary objectives of this project are:

  1. To develop a web-based application that accepts resume uploads in PDF format.
  2. To implement an intelligent parsing engine that extracts text from documents accurately.
  3. To integrate a Generative AI model that understands context, synonyms, and implied skills.
  4. To provide a quantified "Compatibility Score" that ranks candidates objectively.
  5. To generate actionable feedback (Strengths/Weaknesses) that explains why a candidate was shortlisted or rejected.
  6. To ensure a user-friendly experience with immediate feedback and visual data representation.

1.4 Scope of the Project

The current scope of the project focuses on the initial screening phase. It is designed to evaluate individual resumes against specific job roles. The system supports PDF files, which are the industry standard for professional documents. It is built as a modular application, meaning the AI model can be swapped or upgraded without rewriting the entire core logic.

The scope includes:

  • Frontend: A React-based Single Page Application (SPA).
  • Backend: A Python Flask REST API.
  • AI Integration: OpenRouter API connecting to Llama 3.
  • Data Handling: In-memory processing with optional SQLite logging.

1.5 Methodology

The project follows the Agile Software Development Life Cycle (SDLC). This iterative approach allows for continuous refinement of the prompt engineering and UI/UX based on testing feedback. The phases include:

  • Requirement Analysis: Identifying the key metrics for resume evaluation.
  • Design: Architecting the separation between Frontend, Backend, and AI Service.
  • Development: Building the application in two-week sprints.
  • Testing: Verifying AI accuracy and system stability.
  • Deployment: Final release and documentation.

2. LITERATURE SURVEY AND THEORETICAL BACKGROUND

2.1 Artificial Intelligence in Recruitment

The integration of AI in HR, often termed "People Analytics" or "Talent Intelligence," has grown exponentially. Early systems used simple Rule-Based Systems (RBS) where recruiters defined strict "If-Then" rules (e.g., "If Experience < 3 years, then Reject"). While transparent, these systems lacked flexibility.

Modern approaches utilize Machine Learning (ML) and Natural Language Processing (NLP). According to a 2023 report by Gartner, 81% of HR leaders have explored or implemented AI solutions to improve process efficiency. AI can identify patterns in successful hires and apply those patterns to new applicants, moving beyond simple keyword matching to semantic matching.

2.2 Large Language Models (LLMs)

At the core of this project is a Large Language Model. LLMs are deep learning algorithms that can recognize, summarize, translate, predict, and generate text and other content based on knowledge gained from massive datasets.

Meta Llama 3 (8B Instruct): We utilize the Llama 3 model, an open-weights model by Meta. It is a decoder-only transformer architecture. Key advantages for this project include:

  • Contextual Understanding: It understands that "React.js" and "Frontend Development" are related concepts.
  • Instruction Following: It can strictly follow the instruction to "Return ONLY JSON," which is critical for system integration.
  • Zero-Shot Learning: It can evaluate a "Data Scientist" resume without needing to be specifically trained on thousands of Data Science resumes beforehand, relying on its general training data.

2.3 Retrieval Augmented Generation (RAG) Concepts

While not a full RAG system with a vector database, the project utilizes RAG principles by injecting specific context (the Resume Content and the Job Role) into the prompt window of the LLM. This "Grounding" ensures the AI generates answers based solely on the provided document, reducing hallucinations.

2.4 Technology Stack Analysis

2.4.1 Python & Flask

Python is the de-facto language of AI and Data Science due to its rich ecosystem libraries. Flask is a micro-web framework written in Python. It is classified as a microframework because it does not require particular tools or libraries. It has no database abstraction layer, form validation, or any other components where pre-existing third-party libraries provide common functions. This makes it lightweight and perfect for wrapping an AI model in a simple API endpoint.

2.4.2 React.js

React is a free and open-source front-end JavaScript library for building user interfaces based on components. It is maintained by Meta. For this project, React allows us to build a dynamic "Upload -> Processing -> Result" flow without reloading the page, providing a seamless "App-like" feel.

2.4.3 Vite

Vite is a build tool that aims to provide a faster and leaner development experience for modern web projects. It consists of a dev server that serves your source files over native ES modules and a build command that bundles your code with Rollup for production.

3. SYSTEM ANALYSIS

3.1 Feasibility Study

Before development, a feasibility study was conducted to determine the viability of the project.

3.1.1 Technical Feasibility

The project requires integrating a frontend, a backend, and an external API. The team possesses the necessary skills in Python and JavaScript. The availability of powerful public APIs (OpenRouter) eliminates the need for expensive local GPU hardware to run the LLM. Verdict: Feasible.

3.1.2 Economic Feasibility

The development cost involves only developer time and a minimal cost for API tokens. OpenRouter offers competitive pricing (dollars per million tokens). Compared to the cost of human hours saved (recruiter salary), the ROI is positive. Verdict: Feasible.

3.1.3 Operational Feasibility

The system is designed to be intuitive. No special training is required for HR staff. They simply upload a file and read the result. It fits seamlessly into existing workflows. Verdict: Feasible.

3.2 Requirements Specification

3.2.1 Functional Requirements

  1. Resume Upload: The system must accept .pdf files up to 5MB.
  2. Role Specification: The user must be able to input the Job Role/Description.
  3. Text Extraction: The system must strip all formatting and extract pure text from the PDF.
  4. Analysis Generation: The system must generate a 0-100 score and qualitative feedback.
  5. Data Validation: The system must reject non-PDF files and empty inputs.
  6. Result Display: The system must clearly show the "Shortlisted" status based on a configurable threshold (e.g., Score > 60).

3.2.2 Non-Functional Requirements

  1. Performance: The analysis should complete within 15 seconds.
  2. Reliability: The system should handle API timeouts gracefully with user-friendly error messages.
  3. Security: Candidate data must be processed in memory and not exposed publicly. API keys must be secured server-side.
  4. Usability: The UI should be responsive and work on mobile and desktop devices.

3.3 System Environment Checklist

To deploy the application, the following environment is required:

  • Server Side: Python 3.8+, Flask 2.0+, PyPDF2, Requests.
  • Client Side: Node.js 16+ (for build), Modern Web Browser.
  • External: Active Internet Connection (for AI API), OpenRouter API Key.
