System Implementation & Maintenance

AI Resume Shortlister

1. System Implementation

1.1 Technology Stack

Layer Technology Version Purpose
Frontend React + Vite React 18, Vite 5 User interface and single-page application
Backend Flask (Python) Python 3.10+ REST API server
Database SQLite Built-in Local file-based data storage
AI Service Groq API LLaMA 3.1 8B Resume evaluation and scoring

1.3 Module Description

File Module Responsibility
app.py API Server Flask application with REST API endpoints for resume upload, analysis, and history retrieval
llm_client.py AI Client Handles communication with Groq API, sends prompts, parses JSON responses, normalizes scores
database.py Data Layer Database schema definition, connection management, CRUD operations for resume records
utils.py Utilities File text extraction from PDF and plain text uploads
UploadPage.jsx Upload UI Drag-and-drop resume upload interface with role selection
ResultPage.jsx Result UI Displays AI analysis results with score circle, strengths, weaknesses, suggestions
HistoryPage.jsx History UI Shows all previously analyzed resumes with expandable detail rows

1.4 API Endpoints

Method Endpoint Input Output Description
POST /api/analyze-resume FormData: resume (file), role (text) JSON: filename, role, analysis (score, shortlisted, strengths, weaknesses, suggestions) Upload and analyze a resume against a job role
GET /api/resumes None JSON array of all past analysis records Retrieve analysis history from database
GET /health None JSON: { "status": "healthy" } Health check for server monitoring

1.5 Request Flow

User uploads resume + selects role
        |
        v

        |
        v

        |
        v

        |
        v
Groq returns JSON: score, strengths, weaknesses, suggestions
        |
        v
database.py saves result to resumes.db
        |
        v

        |
        v

2. System Maintenance

2.1 Environment Setup

Requirement Purpose Installation
Python 3.10+ Backend runtime python.org/downloads
Node.js 18+ Frontend build tool nodejs.org
Groq API Key AI service access console.groq.com/keys

2.2 Configuration Files

File Variables Description
.env GROQ_API_KEY API key for Groq AI service (required)
.env GROQ_MODEL AI model name (default: llama-3.1-8b-instant)
vite.config.js proxy /api Forwards frontend API calls to Flask backend at port 5000

2.3 Running the Application

Step 1: Set up Python virtual environment
    python -m venv venv
    .\venv\Scripts\activate          (Windows)
    source venv/bin/activate          (Mac/Linux)

Step 2: Install backend dependencies
    pip install -r requirements.txt

Step 3: Configure .env file
    GROQ_API_KEY=your_key_here
    GROQ_MODEL=llama-3.1-8b-instant

Step 4: Start the backend (Terminal 1)
    python app.py
    → Runs on http://localhost:5000

Step 5: Install frontend dependencies
    cd frontend
    npm install

Step 6: Start the frontend (Terminal 2)
    npm run dev
    → Runs on http://localhost:5173

2.4 Database Maintenance

Task Procedure
View database Use any SQLite viewer (DB Browser for SQLite) to open resumes.db
Reset database Delete resumes.db file and restart the backend; it will auto-recreate
Backup database Copy resumes.db file to a backup location
Schema changes Update SCHEMA_SQL in database.py, delete resumes.db, and restart

2.5 Troubleshooting

Issue Cause Solution
Score always 0, suggestions show "AI Service Timeout or Error" Invalid or expired API key Get a new key from console.groq.com and update .env
ECONNREFUSED :5000 Backend not running Start backend with: python app.py
Blank page on frontend Frontend not running Run: cd frontend && npm run dev
"no such column" database error Outdated database schema Delete resumes.db and restart backend
Model decommissioned error AI model no longer available Check available models at console.groq.com and update GROQ_MODEL in .env
